<原文开始>还有一个 log.FatalF() 函数可以接受 % 格式的占位符。</原文结束>

不喜欢 FatalF() 这个名字。最近反而越来越认同 Java 的命名。简洁的命名是好事，就像权力继承时选择最贤明的人是理想情况，但是怎样判定贤明，很容易被误导并反复。"简洁的命名" 也是如此，将它作为理念的话，会失去对命名的可靠预期。
